Compatibility with Windows Server

Please note that Samba4 is not compatible with the latest Windows Server AD Schema. Windows AD schemas up to 2012/2012R2 should work (and are the current defaults).

First steps

There are a number of steps that should be taken prior to use. Please be sure to read them before getting started.

Administer your AD - use RSAT

For most administration tasks of your AD use RSAT (Remote Server Administration Tools).

Backups

Import note re backups: TKLBAM won't backup the AD databases. To recover a problematic Domain Controller, create a new instance and add it to your existing AD domain. Requires an existing AD Domain Controller (at least 2 DCs is recommended best practice.

Further recommendations

See further General recommendations on the TurnKey Domain Controller doc page.

Priting service - CUPS

CUPS is disabled by default - unless you enabled it via interactive firstboot scripts.

  • To enable:
    systemctl enable --now cups.service
  • To disable:
    systemctl disable --now cups.service
